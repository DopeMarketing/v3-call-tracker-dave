# Changelog

All notable changes to V3 Call Tracker Dave will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.1.0] - 2026-03-27

### Added
- Initial project scaffold generated by Project Generator
- Complete database schema with 9 tables:
  - `users` table for user accounts and preferences
  - `integration_configs` table for API keys and settings
  - `call_recordings` table for Fathom recording metadata
  - `transcriptions` table for generated transcription text
  - `calendar_events` table for Google Calendar event data
  - `call_calendar_matches` table linking calls to calendar events
  - `tags` table for custom categorization labels
  - `call_tags` table for many-to-many call tagging
  - `export_jobs` table for background job tracking
- Next.js 15 app router structure with TypeScript
- 16 route stubs from site map:
  - `/` - Main dashboard (public)
  - `/recordings` - Recording list page (public)
  - `/recordings/[id]` - Individual recording page (public)
  - `/transcriptions` - Transcription list page (public)
  - `/transcriptions/[id]` - Individual transcription page (public)
  - `/calendar` - Calendar integration page (public)
  - `/calendar/[event_id]` - Individual calendar event page (public)
  - `/tags` - Tag management page (public)
  - `/tags/[id]` - Individual tag page (public)
  - `/settings` - General settings page (public)
  - `/settings/gdrive` - Google Drive settings (public)
  - `/settings/calendar` - Calendar settings (public)
  - `/settings/transcription` - Transcription settings (public)
  - `/api/fathom/export` - Fathom export API endpoint (public)
  - `/api/transcribe` - Transcription API endpoint (public)
  - `/api/calendar/sync` - Calendar sync API endpoint (public)
- Integration stubs for all required services:
  - Google Drive API client stub
  - Google Calendar API client stub
  - Fathom API integration stub
  - Zapier webhook integration stub
  - Make automation platform stub
  - n8n workflow automation stub
  - Notion knowledge base integration stub
- Supabase configuration with PostgreSQL database
- Tailwind CSS styling setup
- TypeScript strict mode configuration
- Environment variable template (.env.example)
- Vercel deployment configuration
- Complete documentation suite:
  - README.md with setup instructions
  - CLAUDE.md briefing for AI development
  - CHANGELOG.md for version tracking
  - TECHNICAL_DEBT.md for known shortcuts
  - ROADMAP.md for feature planning
- Basic folder structure for scalable development:
  - `/components` for reusable UI components
  - `/lib` for business logic and utilities
  - `/db` for database queries and operations
  - `/actions` for server actions
  - `/types` for TypeScript definitions
- Database migration files with complete schema
- Row Level Security (RLS) policies for data protection
- API route structure for external integrations
